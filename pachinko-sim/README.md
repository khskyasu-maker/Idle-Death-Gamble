# 오사카 난바 1엔 파친코 모의 다이 시뮬레이터

이 프로그램은 일본 여행 시 주 방문 후보인 123難波店과 楽園なんば店의 1엔권/저대여 파친코 기종 체감 확률과 예산 소모를 이해하기 위해 제작된 모의 시뮬레이터입니다. ARROW namBa HIPS와 마루한 나니와는 대화 중 비교·참조용 업장으로만 두며, 로컬 시뮬레이터 점포 선택지에는 넣지 않습니다.

시뮬의 중심 축은 기종 스펙, 실제/가정 회전수, 예산, 중단 규칙입니다. 점포 정보는 레이트, 설치 대수, 보더 환산을 위한 보조 조건이며 점포 추천 순위가 아닙니다.

⚠️ **주의사항** ⚠️
- 본 프로그램은 재미와 예산 감각을 익히기 위한 **조건 비교용 시뮬레이터**입니다.
- 실제 파친코에서의 수익을 보장하지 않으며, 어떠한 공략법이나 필승법도 포함하고 있지 않습니다.
- 실제 매장의 설치 기종 및 스펙은 상시 변경될 수 있으므로 참고용으로만 사용하세요.
- 점포 라인업은 `../data/namba-actual-1yen-lineup.json`을 읽어 구성합니다.
- 라인업 데이터는 DMM 저대여 전체 대수와 로컬 등록 대수를 함께 표시합니다. 현재 로컬 행은 에바/대해물어/기타 수동 확인 후보이며, DMM 전체 1円(1엔)/저대여 전체 기종을 모두 저장했다는 의미가 아닙니다.
- 모든 등록 기종에 시뮬레이션 모델이 있는 것은 아니며, 모델이 없는 기종은 선택지는 아니지만 미지원/관찰 후보로 표시됩니다.
- `stores.py`에서 현재 선택 가능한 점포는 123難波店 1円과 楽園なんば店 1.111円뿐입니다. 선택 가능한 모델은 `ACTIVE_EVA_SIM_MODEL_IDS`의 에바 7개 모델, 大海物語(대해물어), 그리고 `ACTIVE_OTHER_SIM_MODEL_IDS`에 적은 기타 5개 모델(`デジハネP北斗の拳 慈母`, `P Re:ゼロ 鬼がかり 99`, `Pルパン三世 77Sweet`, `e甲鉄城のカバネリ2`, `e東京喰種`)로 제한합니다. HIPS 전용 `P Re:ゼロ season2 129`와 `eゴジラ対エヴァンゲリオン2 超デカゴールド`는 저대여 라인업/참조 모델에는 남을 수 있지만 현 시뮬레이터 선택지에서는 제외합니다. `machine_definitions/`에는 검증용 참조 모델이 더 남아 있을 수 있습니다.
- 기종별 `confidence`가 `low`인 모델은 세부 스펙 확인 전의 보수적 추정값이며, 점수 상한과 주의 문구가 적용됩니다.
- CSV 저장은 사용자가 명시적으로 선택할 때만 로컬 gitignored `results.csv`에 최신 결과로 덮어씁니다. 누적 실행 이력은 남기지 않습니다.
- 공개 공유를 선택하면 `../docs/latest-sim-results.*` 고정 파일 3개(JSON/Markdown/HTML)에 최신 집계표만 덮어씁니다. 시간별 결과 파일은 만들지 않습니다.
- 공개 집계표 일부만 다시 계산할 때는 프로젝트 루트에서 `python3 scripts/publish_sim_results.py --machine-id <id> --merge-existing`을 실행해 기존 표의 해당 기종 행만 교체할 수 있습니다.
- CLI smoke 확인처럼 실제 `docs/`를 건드리면 안 되는 경우에는 `PACHINKO_SIM_PUBLIC_DOCS_DIR=/tmp/sim-export` 환경변수로 export 위치를 바꿀 수 있습니다.
- CLI 출력에서 일본어 기종명, 상태명, 당첨 종류, 스펙 용어가 나오면 한국어 번역을 함께 표시합니다.

## 실행 환경
- Python 3.x
- 별도의 외부 라이브러리 설치가 필요 없는 표준 라이브러리로만 구현되었습니다.

## WSL Ubuntu 실행 방법

1. 터미널(WSL)을 열고 프로젝트 폴더로 이동합니다.
   ```bash
   cd pachinko-sim
   ```
2. 아래 명령어로 시뮬레이터를 실행합니다.
   ```bash
   python3 main.py
   ```
3. 화면의 안내에 따라 매장, 기종, 예산 등을 숫자로 입력하여 시뮬레이션을 진행합니다.

로컬 생성 파일과 구 공개 결과 패턴은 프로젝트 루트에서 dry-run으로 확인할 수 있습니다.

```bash
python3 scripts/clean.py
```

실제 정리는 명시적으로 실행합니다.

```bash
python3 scripts/clean.py --apply
```

## 현재 모델에 반영된 현실 요소
- 보더가 있는 기종은 보더-10/-5/±0/+5/+10 회전율 비교, 보더 미확정 기종은 1,000엔당 50/60/70/80/90/100회전 비교
- 1000엔당 회전수뿐 아니라 200엔/250玉/180玉/보더 마진 환산을 공통 로직으로 처리
- 단일/반복/예산/프로파일/점포 보조 비교 모드에서 `1000엔당 회전수`, `200엔당 회전수`, `250玉당 회전수`, `사용 금액+실제 회전수`, `보더 기준 +/-` 입력을 선택할 수 있습니다.
- 입력 회전수는 평균값으로 취급하고, 실제 세션 회전수는 구슬 수와 헤소 입상 확률의 이항분포 표본으로 흔들리게 처리합니다.
- 5,000엔 / 10,000엔 / 15,000엔 / 20,000엔 예산 비교
- 예산별 평균 체류 시간, P50/P90 체류 시간, 1~9시간 도달률, 9시간 이후 RUSH 종료 정리율, 현금 없이 이어진 시간, 보류 대기 시간, 우타치 시간, 당첨 연출 시간
- 공개 집계표에는 초반 위험 진단용으로 1/2/3시간 도달률, 첫 당첨 전 완전소진률, 첫 당첨까지 평균 현금 사용액을 함께 표시합니다.
- `현금+보유구슬 소진` 방식은 따낸 구슬을 통상 회전에 재사용하되 9시간 이후에는 새 통상 플레이를 시작하지 않고, RUSH/時短(시단)/確変(확변)이 끝나면 정리합니다. 안전용 하드 상한은 11시간입니다.
- 최종 잔류액은 `미사용 현금 + 교환 가능 금액`으로 계산하며, 원래 예산 대비 잔류손익과 구분해 출력합니다.
- 예산을 모두 쓴 뒤 따낸 구슬/RUSH로 이어진 시간과, 현금+구슬이 모두 부족해서 멈춘 비율을 별도 출력합니다.
- 시간 모델은 현금/보유구슬 차감용 순소모와 체류시간용 총발사구슬을 분리합니다. 통상시 ベース(반환)는 바다 25%, 에바/리제로/전락형 배틀 20%를 기본값으로 두고, 총발사구슬은 `순소모 / (1 - ベース)`로 환산합니다.
- 발사 속도는 100발/분, 통상 회전 표시 시간은 6초를 공통 기준으로 두고, 바다/에바/리제로/전락형 배틀 계열별 우타치 초수와 당첨 출옥 속도를 다르게 적용합니다.
- 데이터에 등록된 보더라인을 1000엔당 회전수로 환산해 보더 대비 +/− 회전 표시
- 대여율과 교환율 분리
- 보유 구슬 재사용 여부
- 예산 고정 회전수 방식과 현금+보유구슬 소진 방식 분리
- 우타치(ST/時短(시단)/LT/確変(확변)) 중 평균 구슬 감소
- LT와 LT가 아닌 상위 RUSH를 분리해 `P大海物語5 ブラック(P 대해물어5 블랙)`의 GOLDパールRUSH(골드 펄 러시)를 LT로 잘못 집계하지 않습니다.
- `PA大海物語5 Withアグネス・ラム(PA 대해물어5 With 아그네스 램)`의 ハピネスチャンス(해피니스 찬스)는 비LT 상위모드로 분리하고, ST10회 후 時短110회 구간도 상위모드 당첨 분포로 복귀하게 처리합니다.
