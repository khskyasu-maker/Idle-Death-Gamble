- 요약: 공개 기종 스펙을 상태 전이 모델로 옮긴 뒤, 공개된 예산/회전수/교환율/중단 규칙 조건에서 Monte Carlo로 반복 집계합니다.
- 모델 구성: 기종 스펙은 Machine/Payout 데이터로 보관하고 통상, RUSH/ST, LT, 루프, 転落(전락) 상태를 분리합니다; 에바와 大海物語(대해물어)처럼 흐름이 비슷한 계열은 공통 템플릿을 재사용하고 확률/출옥/전이값만 기종별로 교체합니다; 짧은 챌린지나 時短(시단)은 실제 RUSH 진입 지표와 섞이지 않도록 Payout.counts_as_rush로 분리합니다
- 확률 처리: 통상 회전은 구슬이 ヘソ(헤소)에 들어가는 표본 과정을 거친 뒤, 각 회전을 독립 大当り(대당첨) 시행으로 처리합니다; RUSH/ST/LT는 단일 베르누이 지름길이 아니라 통상→우타치→종료/상위 상태의 전이로 시뮬레이션합니다; 지원되는 기종은 우타치 계속, 転落(전락) 선착순, 残保留(잔보류) 복귀 경로를 기종 계열별로 반영합니다
